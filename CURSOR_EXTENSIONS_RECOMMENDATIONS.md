# Рекомендации по расширениям Cursor

## Анализ проектов

На основе анализа ваших проектов используются следующие технологии:

### Используемые технологии:
- ✅ **JavaScript/Node.js** - основной язык разработки
- ✅ **HTML/CSS** - фронтенд для японской игры
- ✅ **Docker** - контейнеризация (compose.yaml, Dockerfile)
- ✅ **Jest** - тестирование
- ✅ **EJS** - шаблоны (Express.js)
- ✅ **MongoDB/Mongoose** - база данных (в docker-projects)
- ✅ **Markdown** - документация
- ✅ **YAML** - конфигурационные файлы (Docker Compose, Kubernetes)
- ✅ **Express.js** - веб-фреймворк
- ⚠️ **Python** - используется только для `python -m http.server` (можно заменить на `npx http-server`)

### НЕ используются:
- ❌ TypeScript (нет .ts файлов)
- ❌ React/Vue/Angular (упоминаются в документации, но не используются в коде)
- ❌ PHP, Java, C++, Rust, Go
- ❌ PostgreSQL, MySQL (упоминаются в комментариях, но не используются активно)
- ❌ Svelte, Nuxt, Next.js
- ❌ Webpack, Vite (нет конфигурационных файлов)

## Рекомендации

1. **Проверьте установленные расширения**: Откройте Extensions (Ctrl+Shift+X) и проверьте список
2. **Удалите неиспользуемые**: Удалите расширения для языков и фреймворков, которых нет в ваших проектах
3. **Оптимизируйте Python**: Если Python используется только для HTTP сервера, замените на `npx http-server` и удалите Python расширения
4. **Оставьте только необходимое**: Чем меньше расширений, тем быстрее работает Cursor

## Как проверить установленные расширения

1. Откройте панель расширений: `Ctrl+Shift+X`
2. Введите название языка/фреймворка в поиск
3. Проверьте, установлены ли расширения для неиспользуемых технологий
4. Удалите ненужные расширения

## Альтернатива Python HTTP серверу

Вместо `python -m http.server 8000` можно использовать:
```bash
# Node.js вариант (уже есть в проекте)
npx http-server

# Или с указанием порта
npx http-server -p 8000
```

Это позволит полностью убрать зависимость от Python и удалить Python расширения.
